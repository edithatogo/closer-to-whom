# CTW-002 — Maximal local handover implementation

## Objective

Convert the inherited bootstrap into a nationally scoped, machine-readable, agent-optimised research software repository with maximal practical harness engineering, while preserving public-data, aggregate, non-identifiable, and claim-boundary constraints.

## Acceptance criteria

- Git and Conductor state are coherent and versioned.
- Core aggregate simulation, cost, equity, implied-capacity, optimisation, MCDA, DSA, PSA, structural uncertainty, resilience, and VOI foundations exist.
- Polars and Arrow are canonical data interfaces; JAX and Mojo are optional, differential-gated accelerators.
- National service census, demand, clinical pathways, OIA, Healthpoint, and upstream-library work have explicit fail-closed contracts.
- CI/CD covers code quality, property/metamorphic/differential testing, generated-file drift, security, supply chain, docs, dashboard, clean-room reproducibility, benchmarks, mutation/fuzzing, and release receipts.
- A local handover prompt, Git bundle, source ZIP, checksums, SBOM, release manifest, and validation report are produced.
- No scientific conclusion is drawn from synthetic fixtures.
